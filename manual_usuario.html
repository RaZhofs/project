1. Introducción

Convexa es un sistema web de gestión de eventos diseñado para administrar equipos, tareas, bitácoras y presupuestos desde una sola plataforma. Está construido con una arquitectura de tres capas:

  • Base de datos: SQL Server (cualquier edición, incluyendo Express gratuita)
  • Backend: Node.js + Express + Sequelize (API REST)
  • Frontend: React + Vite + Tailwind CSS v4

El sistema contempla dos roles: Administrador, que gestiona eventos completos, y Colaborador, que visualiza y actualiza sus tareas asignadas y puede participar en la bitácora de cada evento.

3. Guía de Instalación

3.1 Clonar el repositorio

Abre una terminal (PowerShell o CMD) en la carpeta donde quieres instalar el proyecto:

git clone https://github.com/<tu-usuario>/convexa.git
cd convexa

La estructura de carpetas que encontrarás es:

convexa/
├── backend/          ← API Node.js + Express
│   ├── migrations/   ← Scripts SQL a ejecutar en SSMS
│   ├── seeders/      ← Datos iniciales de prueba
│   └── src/
├── frontend/         ← Aplicación React
└── db/               ← Schema completo (referencia)

3.2 Configurar SQL Server

  1. Habilitar autenticación SQL
    En SSMS, clic derecho sobre el servidor → Propiedades → Seguridad → selecciona "Modo de autenticación de SQL Server y Windows". Reinicia el servicio SQL Server.
  2. Crear usuario SQL
    En SSMS ejecuta: 
    CREATE LOGIN conglomerado_user WITH PASSWORD = 'ConglomPass2026!';
CREATE DATABASE conglomerado_db;
USE conglomerado_db;
CREATE USER conglomerado_user FOR LOGIN conglomerado_user;
ALTER ROLE db_owner ADD MEMBER conglomerado_user;
  3. Ejecutar las migraciones
    En SSMS, abre y ejecuta cada archivo de la carpeta backend/migrations/ en orden numérico:
    ArchivoCrea
    001_create_roles.sqlTabla ROLES
    002_create_usuarios.sqlTabla USUARIOS
    003_create_administradores.sqlTabla ADMINISTRADORES
    004_create_tipos_evento.sqlTabla TIPOS_EVENTO
    005_create_eventos.sqlTabla EVENTOS
    006_create_tareas.sqlTabla TAREAS
    007_create_colaboradores.sqlTabla COLABORADORES
    008_create_equipo_colaboradores.sqlTabla EQUIPO_COLABORADORES
    009_patch_tareas_fk_responsable.sqlFK responsable en TAREAS
    010_create_bitacora.sqlTabla BITACORA
    011_create_presupuesto.sqlTabla PRESUPUESTO_ITEMS
    012_create_invitados_rsvp.sqlTabla INVITADOS_RSVP (registro de asistentes)
    013_create_restricciones_invitado.sqlTabla RESTRICCIONES_INVITADO
    ⚠️ Importante: Ejecuta los scripts en orden estricto (001 → 013). Cada tabla tiene dependencias de FK con las anteriores.

3.3 Instalar y configurar el Backend

  1. Instalar dependencias 
    cd backend
npm install
  2. Crear el archivo de entorno
    Crea el archivo backend/.env con el siguiente contenido (ajusta los valores si tu instancia SQL Server es diferente): 
    DB_HOST=127.0.0.1
DB_PORT=1433
DB_NAME=conglomerado_db
DB_USER=conglomerado_user
DB_PASSWORD=ConglomPass2026!
PORT=3001
    ✅ SQL Server Express con instancia nombrada: Si usas una instancia con nombre (ej. SQLEXPRESS), agrega:
    DB_INSTANCE=SQLEXPRESS
    y deja DB_PORT sin valor o comenta esa línea.
  3. Poblar los datos iniciales
    Este comando crea los roles, tipos de evento y los dos usuarios de demostración: 
    npm run seed
    Deberías ver: 
    ✓ Roles sembrados
✓ Tipos de evento sembrados (8 tipos)
✓ Admin demo creado.  → id_administrador: 1
✓ Colaborador demo creado. → id_colaborador: 1
  4. Iniciar el servidor 
    npm run dev
    Resultado esperado: Servidor corriendo en puerto 3001

3.4 Instalar y ejecutar el Frontend

  1. Instalar dependencias 
    cd ../frontend
npm install
  2. Iniciar la aplicación 
    npm run dev
    La aplicación quedará disponible en http://localhost:5173
✅ Flujo de inicio rápido (resumen)
Terminal 1 → cd backend && npm run dev
Terminal 2 → cd frontend && npm run dev
Navegador → http://localhost:5173

6. Vista Administrador

6.1 Pantalla de Login

Al ingresar a http://localhost:5173, el sistema muestra el formulario de acceso. Ingresa el correo y contraseña del administrador y haz clic en Ingresar. El sistema redirige automáticamente al Dashboard si el rol es Administrador, o a Mis Tareas si es Colaborador.

6.2 Dashboard — Lista de Eventos

El dashboard muestra todos los eventos registrados en el sistema. Dispone de dos modos de visualización que se alternan con los botones de la barra superior derecha:

Vista Tarjetas (Cards)

Cada tarjeta representa un evento y contiene la siguiente información distribuida en secciones:

  • Borde superior de color — indica el estado del evento:
    • 🟡 Amarillo → Planificación
    • 🔵 Azul → Montaje
    • 🟢 Verde → Ejecución
    • ⚪ Gris → Finalizado
    • 🔴 Rojo → Cancelado
  • Nombre del evento — enlace clicable que lleva al detalle del evento
  • Número ID — identificador interno (esquina superior derecha de la tarjeta)
  • Badges de estado y modalidad — píldoras de color con el estado actual y la modalidad
  • Cuadrícula de metadatos — Tipo de evento, Aforo máximo, Fecha de inicio, Fecha de término
  • Ubicación — ícono de pin con la dirección (si fue ingresada)
  • Pie de tarjeta — nombre del administrador responsable + botones Editar y Eliminar

Vista Tabla

Muestra todos los eventos en una tabla con columnas: Evento (enlace), Tipo, Administrador, Inicio, Término, Aforo, Estado/Modalidad (badges) y Acciones (Editar / Eliminar). Útil cuando hay muchos eventos y se necesita comparar datos rápidamente.

6.3 Crear un Nuevo Evento

7. Detalle de Evento — Las Cuatro Pestañas

Al hacer clic en el nombre de cualquier evento (desde tarjeta o tabla), accedes a la página de detalle. En la parte superior se muestra un resumen con nombre, badges de estado y modalidad, tipo, fechas, aforo y ubicación. Debajo, una barra de navegación con cuatro pestañas:

Pestaña 1 — Equipo

¿Qué muestra?

Lista de todos los colaboradores asignados a este evento. Cada fila muestra nombre completo, correo, fecha de asignación y nivel de permiso (Lectura / Edición / Administración).

Acciones disponibles

  • Asignar colaborador — botón en la esquina superior derecha. Abre un modal con dos campos:
    • Colaborador: lista de colaboradores aún no asignados al evento
    • Nivel de permiso: Lectura, Edición o Administración
  • Quitar — botón rojo en cada fila para remover al colaborador del evento
💡 Nota: El botón "Asignar colaborador" aparece deshabilitado si todos los colaboradores del sistema ya están asignados al evento, o si no existe ningún colaborador en la BD.

Pestaña 2 — Tareas

¿Qué muestra?

Tabla con todas las tareas creadas para este evento. Columnas: Tarea (título + descripción), Responsable (colaborador asignado), Prioridad, Estado, Fecha límite y Acciones.

Acciones disponibles

  • + Nueva tarea — abre modal con los campos:
    • Título Requerido
    • Descripción Opcional
    • Responsable (colaboradores asignados al evento)
    • Prioridad: Alta / Media / Baja
    • Fecha límite Requerido
  • Editar — permite cambiar el responsable y el estado de la tarea (Pendiente / En Proceso / Completada)
  • Eliminar — borra la tarea con confirmación previa
✅ Flujo correcto: Primero asigna colaboradores en la pestaña Equipo, luego crea tareas asignándoles esos colaboradores como responsables. Solo así aparecerán en el panel del colaborador.

Pestaña 3 — Bitácora

¿Qué muestra?

Feed cronológico (más reciente primero) de todas las entradas registradas para el evento. Cada entrada muestra: tipo (badge de color), nombre del autor, fecha y hora, y el contenido.

Tipos de entrada y colores

  • ⚪ Nota — observación general (gris)
  • 🟢 Avance — progreso reportado (verde)
  • 🟡 Alerta — situación a atender (amarillo)
  • 🔴 Incidencia — problema registrado (rojo)

Acciones disponibles

  • Selecciona el tipo de entrada con los botones pill en la parte superior del formulario, escribe el contenido en el área de texto y haz clic en Publicar.
💡 Compartido: Tanto el administrador como los colaboradores asignados pueden publicar y leer las entradas de la bitácora del evento.

Pestaña 4 — Presupuesto

¿Qué muestra?

Panel de control financiero del evento con:

  • 4 chips de resumen en la parte superior:
    • 🟢 Ingresos estimados — suma de todos los ítems tipo Ingreso
    • 🔴 Egresos estimados — suma de todos los ítems tipo Egreso
    • ⚖️ Balance — Ingresos − Egresos (verde si positivo, rojo si negativo)
    • 🔵 Total ejecutado — suma de los montos reales registrados
  • Tabla de ítems con columnas: Descripción, Categoría, Tipo (Ingreso/Egreso), Monto estimado, Monto real, Estado y Acciones.

Acciones disponibles

  • + Nuevo ítem — modal con los campos:
    • Descripción Requerido
    • Categoría: Infraestructura / Marketing / Personal / Catering / Tecnología / Logística / Otros
    • Tipo: Ingreso / Egreso
    • Monto estimado Requerido
    • Monto real Opcional — completar cuando se ejecute el gasto
    • Estado: Pendiente / Aprobado / Ejecutado
  • Editar — modificar cualquier campo del ítem
  • Eliminar — borra el ítem con confirmación previa

8. Vista Colaborador

Al iniciar sesión con credenciales de colaborador, el sistema redirige a /mis-tareas. Esta vista tiene su propia barra de navegación con dos pestañas:

Pestaña 1 — Mis Tareas

¿Qué muestra?

  • 4 chips estadísticos: Total de tareas, Pendientes, En proceso y Completadas
  • Filtros de estado: botones pill para filtrar por Todas / Pendiente / En Proceso / Completada
  • Tarjetas de tarea — una por cada tarea asignada al colaborador. Cada tarjeta muestra:
    • Barra de color lateral según prioridad (🔴 Alta, 🟡 Media, 🟢 Baja)
    • Título y descripción de la tarea
    • Badge de estado actual (Pendiente / En Proceso / Completada)
    • Badge de prioridad
    • Nombre del evento al que pertenece la tarea
    • Fecha límite (en rojo si está vencida)
    • Botón "Siguiente estado" para avanzar el estado cíclicamente
💡 Nota: Solo aparecen aquí las tareas donde el colaborador fue designado explícitamente como Responsable por el administrador.

Pestaña 2 — Bitácora

¿Qué muestra?

Si el colaborador está asignado a un solo evento, muestra directamente la bitácora de ese evento. Si está asignado a múltiples eventos, aparecen botones pill para seleccionar cuál evento ver.

El colaborador puede publicar entradas (Nota / Avance / Alerta / Incidencia) y ver todas las entradas publicadas por el administrador y los demás colaboradores del mismo evento.

9. Flujo de Trabajo Recomendado

Para sacar el máximo provecho del sistema con un nuevo evento, sigue estos pasos:

  1. Crear el evento
    Dashboard → "+ Nuevo evento" → completar formulario → Guardar.
  2. Asignar el equipo
    Clic en el nombre del evento → pestaña Equipo → "Asignar colaborador" → seleccionar persona y nivel de permiso → Asignar. Repetir por cada integrante.
  3. Crear las tareas
    Pestaña Tareas → "+ Nueva tarea" → completar título, prioridad, fecha límite y seleccionar un responsable del equipo ya asignado → Crear tarea.
  4. El colaborador gestiona sus tareas
    El colaborador ingresa con su cuenta → pestaña Mis Tareas → usa el botón "Siguiente estado" para avanzar cada tarea (Pendiente → En Proceso → Completada).
  5. Registrar avances en la bitácora
    Tanto el administrador (pestaña Bitácora del evento) como el colaborador (pestaña Bitácora en su panel) pueden publicar notas, avances, alertas e incidencias que quedan registradas cronológicamente.
  6. Gestionar el presupuesto
    Pestaña Presupuesto → "+ Nuevo ítem" → registrar ingresos y egresos estimados. A medida que se ejecutan los gastos, editar cada ítem para agregar el monto real y cambiar su estado a Ejecutado.

10. Solución de Problemas Comunes

ProblemaCausa probableSolución
El backend no conecta a la BD Credenciales incorrectas en .env o autenticación SQL deshabilitada Verificar DB_USER y DB_PASSWORD. Asegurarse de que el modo de autenticación mixto esté habilitado en SQL Server.
El dropdown de Tipo de evento aparece vacío No se corrió el seeder de tipos de evento Ejecutar npm run seed en la carpeta backend.
El botón "Asignar colaborador" está deshabilitado No hay colaboradores en la BD o todos ya están asignados Ejecutar npm run seed para crear el colaborador demo, o agregar colaboradores manualmente.
Las tareas no aparecen en "Mis Tareas" Las tareas no tienen responsable asignado (id_responsable es NULL) En la pestaña Tareas del admin, usar el botón Editar de cada tarea y asignar un responsable.
Error "Invalid object name 'BITACORA'" La migración 010 no fue ejecutada en SSMS Abrir backend/migrations/010_create_bitacora.sql en SSMS y ejecutar.
Error "Invalid object name 'PRESUPUESTO_ITEMS'" La migración 011 no fue ejecutada en SSMS Abrir backend/migrations/011_create_presupuesto.sql en SSMS y ejecutar.
El frontend no carga / error CORS El backend no está corriendo Verificar que npm run dev esté activo en la carpeta backend y que responda en localhost:3001.
Instancia SQL con nombre (SQLExpress) no conecta Falta la variable DB_INSTANCE en .env Agregar DB_INSTANCE=SQLEXPRESS (o el nombre de tu instancia) al archivo .env.
